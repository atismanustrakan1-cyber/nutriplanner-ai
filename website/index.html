Overview

What it is

A nutrition management project built in phases. We start with the core logic in C to keep it fast, portable, and easy to test.

Why C first

The core engine becomes reusable across platforms (Python wrapper, Java client, mobile later) without rewriting the math.

How we build

CMake + unit tests + GitHub Actions. Every change should compile and pass tests before merging.

Daily Log

Meals Today

  • Daily calorie target (loss / maintain / gain)
  • Macro targets (protein/carbs/fat)
  • Meal logging for a day
  • Totals + remaining amounts vs target
  • Unit tests for public API
  • CLI demo for quick testing

Calorie Tracking / Macro Tracking

  • Meal schedule generator
  • Budget-friendly grocery lists
  • Nearby store prices + stock
  • Dietary restrictions & substitutions
  • Camera feature (CV calorie estimate)

Build & Run

From the repo root (the folder that contains c_core/):

cmake -S c_core -B c_core/build
cmake --build c_core/build
ctest --test-dir c_core/build --output-on-failure
./c_core/build/cli_demo
